#### نقش‌ها  
در پروژه، پنج نقش اصلی وجود دارد:  
1. مدیر (Admin)  
2.  (SuperAdmin)  
3. مشتری (Customer)  
4. آرایشگر (Barber)  
5. صاحب آرایشگاه (BarberShopOwner)  

.هر شخص باید تمامی موجودیت‌های مربوط به نقش خود را اضافه کند. نیازی به رعایت اصول شی‌گرایی نیست؛ هدف اصلی ساخت دقیق موجودیت‌ها بر اساس فیگما است
 .فقط یه کلاس لیب بسازید با موجودیت های داخل فیگما  هیچ موجودیتی اضافه یا حذف نشه در غیر این صورت تسک رد میشه
 .هر شخص باید موجودیت‌های مرتبط با نقش خود را در کلاس‌های جداگانه پیاده‌سازی کند  
 .اسم کلاس لیب ها باید متناسب با تسک باشد

  public string CustomerFirstName { get; set; }
